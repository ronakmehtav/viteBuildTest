This is the video might get outdated but here it is anyway Video on how to deploy vite you will have to check the vite docs

Steps:

  1. You have to init the project on Github and push the code.
  2. if vite config is not present create vite config with the file name vite.config.js and put this code inside. import { defineConfig } from 'vite'; export default defineConfig(({ command }) => { if (command === 'build') { return { base: '/${repoName}/', }; } else { return { base: '/', }; } }); So as to pass the base for deployment which you ll need cause else the path will start from the domain itself and not from you repo as github wants for content github urls are https://{Username}-github.io/{repoName}.
  3. run pnpm run build pnpm is a package manager you can use your package manager for example npm,etc .
  4. This should generate the dist folder in the root.
  5. run git add dist -f the .gitignore file ignores the dist folder by default and commit the change by running git commit -m "adding the dist".
  6. run git subtree push --prefix dist origin gh-pages we are create gh-pages branch for our page to show.
  7. if you modify the code and want to push the change you ll have to run all the above commands.
  8. If you run into the error: failed to push some refs to "..." I dont better way then to just delete the branch of gh-pages by running git branch -D local_branch_name git push remote_name -d remote_branch_name if you find a better way to this raise a pr.
